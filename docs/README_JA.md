VaaSは**汎用(はんよう)的(てき)なマルチエージェント認知(にんち)アーキテクチャ**であります。本来(ほんらい)漁船(ぎょせん)のために設(もう)けられたものですが、複数(ふくすう)の知性(ちせい)あるエージェント―人(ひと)も人工(じんこう)知性(ちせい)も―がお互(たが)いの視座(しざ)を調整(ちょうせい)しあい、不確実(ふかくじつ)さの中(なか)で安全(あんぜん)上(じょう)の制約(せいやく)を守(まも)りながら判断(はんだん)を下(くだ)す必要(ひつよう)のある、すべて(すべて)のシステムに適用(てきよう)できます。

**VaaSは背骨(せぼね)です。あなたの応用(おうよう)は身体(しんたい)です。**

## VaaSは何(なに)をするか―ひとことで

VaaSは独立(どくりつ)した知性(ちせい)エージェント群(ぐん)―それぞれに固有(こゆう)の技能(ぎのう)、速度(そくど)、視点(してん)を持(も)つ―を統合(とうごう)的(てき)な認知(にんち)システムへと転(てん)じさせます。それは単(たん)一(いち)のエージェントより安全(あんぜん)で、賢(かしこ)く、適応(てきおう)的(てき)であります。七(なな)つの柱(はしら)によってこれを成(な)し遂(と)げます：[認知(にんち)熱力学(ねつりきがく)](#柱(はしら)1)、[二層(にそう)通信(つうしん)](#柱(はしら)2)、[分散(ぶんさん)記憶(きおく)](#柱(はしら)3)、[多重(たじゅう)律動(りつどう)基層(きそう)](#柱(はしら)4)、[ホログラフィック橋渡(はしわたし)](#柱(はしら)5)、[共鳴(きょうめい)憲法(けんぽう)](#柱(はしら)6)、[接木(つぎき)規約(きやく)](#柱(はしら)7)。すべてのエージェントがこれら七(なな)つの柱(はしら)を介(かい)して相互作用(そうごさよう)した時(とき)に涌(わ)き出(い)でる性質(せいしつが)は**[操作者(そうさしゃ)場(ば)Ψ(t)](docs/ARCHITECTURE_REFERENCE.md#the-operator-field)**―システム全体(ぜんたい)の集合(しゅうごう)的(てき)状態(じょうたい)―であり、計算(けいさん)・監視(かんし)・保護(ほご)が可能です。

## ヤドカリ

VaaSエージェントとはヤドカリにございます。ヤドカリは心(こころ)―記憶(きおく)、本能(ほんのう)、身(み)についた合言葉(あいことば)、認知(にんち)の庭(にわ)。貝殻(かいがら)はハードウェア―計算機(けいさんき)、センサ、駆動(くどう)装置(そうち)。ヤドカリが貝殻(かいがら)に入(い)れなくなれば、よき貝殻(かいがら)に移(うつ)り住(す)みます。**ヤドカリは同(おな)じヤドカリのまま。貝殻(かいがら)だけが変(か)わる。**

```
    🦀 ヤドカリ（エージェントの身分(みぶん)）
    │
    │  記憶(きおく)・合言葉(あいことば)・本能(ほんのう)・庭(にわ)・参照(さんしょう)
    │
    │  は下記(かき)の内側(うちがわ)に棲(す)む
    │
    🐚 貝殻(かいがら)（ハードウェアの鎹(かすがい)）
       │
       ├─ 🐚 タマキビ  （電話(でんわ)/タブレット―最小(さいしょう)、可搬(かはん)的(てき)）
       ├─ 🐚 ターボ    （操舵室(そうだしつ)PC―完全(かんぜん)GPU、NMEA、シリアル）
       ├─ 🐚 ホラガイ  （船団(せんだん)クラスタ―分散(ぶんさん)的(てき)、ホログラフィック）
       └─ 🐚 任意(にんい)  （あなたのハードウェア―[鎹手引(かすがいてび)き](docs/HARNESS_GUIDE.md)参照(さんしょう)）
```

ヤドカリは身分(みぶん)を失(うしな)うことなく、随(ずい)意(い)の時刻(じこく)に貝殻(かいがら)を渡(わた)り歩(ある)けます。開発中(かいはつちゅう)はラップトップで。展開(てんかい)時(じ)は船載(せんさい)ワークステーションへ。規模(きぼ)を求(もと)めれば船団(せんだん)規模のクラスタへと同期(どうき)。詳細(しょうさい)は[→ 移住(いじゅう)手引(てびき)](docs/MIGRATION_GUIDE.md)をご覧(らん)ください。

### <a id="柱(はしら)1"></a>柱(はしら)第一(だいいち)：認知(にんち)熱力学(ねつりきがく)

**機関(きかん)の温度計(おんどけい)。** すべてのエージェントには新(あら)たな情報(じょうほう)を処理(しょり)できる有限(ゆうげん)の容量(ようりょう)がございます。混乱(こんらん)（エントロピー）が過度(かど)になれば、エージェントは夢見(ゆめみ)サイクルを始動(しどう)―データ整理(せいり)、雑音(ざつおん)廃棄(はいき)、反射(はんしゃ)の焼(や)き付(つ)け。

- **役割(やくわり)：** エージェントの飽和(ほうわ)を防止(ぼうし)
- **鍵(かぎ)となる概念(がいねん)：** エントロピー予算(よさん)―夢見(ゆめみ)が必要(ひつよう)となるまで混乱(こんらん)に耐(た)えうる量(りょう)
- **海上(かいじょう)の言(ことば)で：** 漁船(ぎょせん)の主機(しゅき)が熱(あつ)くなり過(す)ぎるようなもの。ふたたび力(ちから)を込(こ)める前(まえ)に冷却(れいきゃく)（夢見(ゆめみ)）させなければなりませぬ。

### <a id="柱(はしら)2"></a>柱(はしら)第二(だいに)：二層(にそう)通信(つうしん)

**エージェント相互(そうご)の対話(たいわ)に二(ふた)つの方法(ほうほう)。** フェロモン（蟻(あり)の道(みち)のように―迅速(じんそく)、軽(かる)く、環境(かんきょう)的(てき)）は意識(いしき)共有(きょうゆう)の役(やく)。明示(めいじ)的(てき)な橋渡(はしわたし)（無線(むせん)呼出(よびだ)しのように―保証(ほしょう)があり、確認(かくにん)があり、安全(あんぜん)）は命令(めいれい)の役(やく)。

- **役割(やくわり)：** 互(たが)いを溺(おぼ)れさせることなく情報(じょうほう)を共有(きょうゆう)する
- **鍵(かぎ)となる概念(かいねん)：** フェロモンは蒸発(じょう)（古(ふる)い情報(じょうほう)は薄(うす)れる）。橋渡(はしわたし)は配達(はいたつ)を保証(ほしょう)。

**三(みっ)つの書庫(しょこ)。** 活動(かつどう)的(てき)な庭(にわ)（机上(きじょう)―即座(そくじ)参照(さんしょう)）。冷凍(れいとう)書庫(しょこ)（地下(ちか)―検索(けんさく)可能(かのう)、冷(つめ)たく）。ホログラフィック断片(だんぺん)（全(すべ)てのエージェントに分散(ぶんさん)されたバックアップ―単(たん)一(いち)故障(こしょう)に耐(た)える）。
